# DarkSword-RCE 项目演示配置指南

## 当前状态
❌ **无法直接演示** - 需要配置本地服务器和修改硬编码地址

## 需要修改的配置

### 1. 服务器地址统一配置

需要修改以下文件中的服务器地址：

#### 文件 1: `DarkSword-RCE/frame.html` (第7行)
```javascript
// 修改前
src="http://192.168.1.27/assets/rce_loader.js?"

// 修改后 (假设本地服务器运行在 127.0.0.1:8088)
src="http://127.0.0.1:8088/rce_loader.js?"
```

#### 文件 2: `DarkSword-RCE/rce_loader.js` (第8行)
```javascript
// 修改前
var localHost = "http://43.156.104.233/"

// 修改后
var localHost = "http://127.0.0.1:8088/"
```

#### 文件 3: `DarkSword-RCE/rce_loader.js` (第27行)
```javascript
// 修改前
window.location.href = "http://43.156.104.233/404.html";

// 修改后
window.location.href = "http://127.0.0.1:8088/404.html";
```

#### 文件 4: `DarkSword-RCE/scan.js` (第2行)
```javascript
// 修改前
const server_url="https://static.cdncounter.net";

// 修改后
const server_url="http://127.0.0.1:8088";
```

### 2. 服务器端配置

#### 安装依赖
```bash
cd server
npm install
```

#### 启动服务器
```bash
node server.js
```

服务器将在 `http://127.0.0.1:8088` 启动

### 3. 提供静态文件服务

当前 `server.js` 只提供 API 接口，需要提供静态文件服务：

在 `server/server.js` 中添加：
```javascript
// 添加静态文件服务
const path = require('path');
app.use('/assets', express.static(path.join(__dirname, '../DarkSword-RCE')));
app.use(express.static(path.join(__dirname, '../DarkSword-RCE')));
```

### 4. 演示环境要求

- **Node.js**: v14 或更高版本
- **浏览器**: 现代浏览器（Chrome/Firefox/Safari）
- **操作系统**: Windows/Mac/Linux
- **注意**: 此项目仅用于安全研究和教育目的

### 5. 演示步骤

1. 修改所有硬编码的服务器地址为 `http://127.0.0.1:8088`
2. 在 `server` 目录运行 `npm install`
3. 修改 `server.js` 添加静态文件服务
4. 运行 `node server.js` 启动服务器
5. 在浏览器中访问 `http://127.0.0.1:8088/index.html`
6. 观察服务器控制台日志

## 自动化配置脚本

我已为您创建了一个配置脚本 `setup-demo.js`，可以自动完成大部分配置工作。

运行方式：
```bash
node setup-demo.js
```

这将：
1. 备份原始文件
2. 修改服务器地址
3. 更新 server.js 配置
4. 安装依赖
